# microdot
Uso promedio entre 80000 y 100000 en memoria, con picos de 110000 y minimas de 30000

# Nanoweb
Promedio entre 100000 y 110000, con picos de 120000 y minimas de 80000

# Picoweb
Promedio bastante solido casi sin variar, aunque tiene respuestas lentas cuando se hace muchas peticiones, con picos de 120000 y minimas de 100000
pero se mantiene estable en 106000

## Benchmark HTTP controlado (ESP32 AP 192.168.4.1)

Script usado: `tools/http_bench.py`

### Microdot (main.py)
- GET `/api/state` secuencial (120): avg 154.97 ms, p50 151.73 ms, p95 174.88 ms, errores 0
- POST `/move` secuencial (80): avg 197.68 ms, p50 195.20 ms, p95 216.56 ms, errores 0
- GET `/api/state` rafaga (8 workers, 120): avg 1207.09 ms, p50 1226.09 ms, p95 1260.75 ms, errores 0
- `mem_free` (80 muestras): promedio 55955, min 19520, max 92032

### Nanoweb (examples/main_nanoweb.py)
- GET `/api/state` secuencial (120): avg 134.76 ms, p50 131.85 ms, p95 153.16 ms, errores 0
- POST `/move` secuencial (80): avg 174.69 ms, p50 172.07 ms, p95 190.71 ms, errores 0
- GET `/api/state` rafaga (8 workers, 120): avg 1011.53 ms, p50 1022.72 ms, p95 1100.66 ms, errores 0
- `mem_free` (80 muestras): promedio 104758, min 90384, max 119728

### Tinyweb (examples/main_tinyweb.py)
- GET `/api/state` secuencial (120): avg 157.15 ms, p50 153.04 ms, p95 172.32 ms, errores 0
- POST `/move` secuencial (80): avg 209.73 ms, p50 205.68 ms, p95 235.19 ms, errores 0
- GET `/api/state` rafaga (8 workers, 120): avg 1288.13 ms, p50 1339.91 ms, p95 1356.80 ms, errores 3
- `mem_free` (80 muestras): promedio 110608, min 104272, max 112464

### Lectura rapida
- Microdot quedo en punto medio de latencia, pero con menor `mem_free` promedio y minima mas baja.
- Nanoweb fue mas rapido en latencia secuencial y rafaga durante esta corrida.
- Tinyweb mostro RAM mas alta y estable en promedio durante las muestras.
- Bajo rafaga, Tinyweb presento 3 errores y Nanoweb 0 en este set.
